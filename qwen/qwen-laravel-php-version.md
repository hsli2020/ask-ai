# 列出Laravel v10+各个版本所要求的最低php版本号

以下是 Laravel `v10+` 各**大版本**对应的最低 PHP 版本要求。在同一个大版本内（例如 `10.0.0` 到 `10.48.x`），最低 PHP 要求始终保持一致，不会因小版本或补丁版本而改变。

| Laravel 大版本 | 最低 PHP 版本要求 | 发布日期 | 备注说明 |
|:---|:---|:---|:---|
| **Laravel 10.x** | `PHP 8.1.0` | 2023年2月 | 全系列要求 `^8.1`，支持至 PHP 8.3 |
| **Laravel 11.x** | `PHP 8.2.0` | 2024年3月 | 全系列要求 `^8.2`，官方推荐使用 PHP 8.3+ |
| **Laravel 12.x** | `PHP 8.2.0` | 2025年3月 | 最低仍为 `^8.2`，但深度优化了 PHP 8.3/8.4 特性 |
| **Laravel 13.x** | *预计 `PHP 8.3.0`* | 2026年3月（已发布/即将发布） | 具体以该版本 `composer.json` 为准，通常每年 3 月按周期发布 |

📌 **重要说明：**
1. **版本内一致性**：Laravel 遵循语义化版本控制，`10.x` 下的所有次要版本（如 `10.0`、`10.15`、`10.48`）**最低 PHP 要求完全相同**。
2. **如何精确验证**：任何版本的最低 PHP 要求均可通过查看该版本仓库的 `composer.json` 文件确认：
   ```json
   "require": {
       "php": "^8.2"  // 表示最低 8.2.0，最高兼容 8.x 最新版
   }
   ```
3. **生产环境建议**：
   - PHP 8.1 已于 `2023-11-25` 结束主动支持（EOL），仅接收安全补丁至 `2024-11-25`。
   - PHP 8.2 将于 `2025-12-08` EOL。
   - 强烈建议在 Laravel 11/12+ 项目中使用 **PHP 8.3 或 PHP 8.4**，以获得更好的性能、类型系统及安全更新。
4. **官方文档参考**：  
   🔗 https://laravel.com/docs/12.x/installation#server-requirements

如需查询某个具体补丁版本（如 `v10.48.12`）的依赖，可提供完整版本号，我可为您核对对应的 `composer.lock` 约束。
