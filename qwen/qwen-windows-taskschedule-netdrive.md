# 在windows下，我创建了一个网络映射驱动器，然后编写了一个脚本，被task scheduler调度执行，定期向此网络映射驱动器中拷贝一些文件。我手工运行了脚本，能正常工作，但放入task scheduler自动执行时，总是不成功，文件没有被拷贝到网络映射驱动器，请问这是什么原因，如何解决？

这是一个在 Windows 任务计划程序（Task Scheduler）中非常经典的问题。

### 🔍 核心原因

**网络映射驱动器（如 `Z:` 盘）是基于“用户交互式会话”（User Interactive Session）的。**
当你手工运行脚本时，你处于登录状态，系统为你挂载了映射驱动器，所以脚本能找到 `Z:` 盘。
但是，当任务计划程序在后台自动执行任务时（尤其是选择了“不管用户是否登录都要运行”时），它运行在一个**非交互式会话**或系统会话中。在这个后台会话里，**你之前手动创建的网络映射驱动器是不存在的**。此外，如果任务是以 `SYSTEM` 账户运行的，该账户默认没有网络访问权限。

---

### 🛠️ 解决方案（按推荐程度排序）

#### 方案一：使用 UNC 路径代替映射驱动器字母（🌟 最推荐、最稳定）

这是微软官方推荐的最佳实践。不要在脚本中使用 `Z:\folder\file.txt`，而是直接使用网络的 UNC 路径 `\\服务器名或IP\共享文件夹名\folder\file.txt`。

- **Batch 示例**:
  ```bat
  copy "C:\local\file.txt" "\\192.168.1.100\ShareName\folder\file.txt"
  ```
- **PowerShell 示例**:
  ```powershell
  Copy-Item -Path "C:\local\file.txt" -Destination "\\192.168.1.100\ShareName\folder\file.txt" -Force
  ```

**前提条件**：运行该计划任务的 Windows 账户，必须对目标 UNC 路径拥有**读写权限**。

#### 方案二：在脚本内部动态映射网络驱动器

如果你的脚本或调用的第三方程序强制要求必须使用驱动器字母（如 `Z:`），你可以在脚本的开头动态创建映射，并在结尾断开。

- **Batch 示例**:
  ```bat
  @echo off
  :: 1. 动态映射驱动器 (替换为你的服务器IP、共享名、用户名和密码)
  net use Z: \\192.168.1.100\ShareName /user:YourDomain\YourUsername YourPassword

  :: 2. 执行拷贝操作
  copy "C:\local\file.txt" "Z:\folder\file.txt"

  :: 3. 断开映射 (清理现场)
  net use Z: /delete /y
  ```

*(注意：在脚本中明文写密码存在安全风险，请确保该脚本文件的 NTFS 权限仅允许管理员或运行任务的用户读取。)*

#### 方案三：正确配置任务计划程序（关键设置）

无论你使用方案一还是方案二，任务计划程序的配置都必须正确，否则依然会因权限问题失败：

1. **不要使用 `SYSTEM` 账户**：在任务的“常规”选项卡中，点击“更改用户或组”，选择一个**具有目标网络共享读写权限的域账户或本地账户**（不要用 `NT AUTHORITY\SYSTEM`）。
2. **选择“不管用户是否登录都要运行”**：这样任务才能在后台执行。
3. **勾选“使用最高权限运行”**：如果你的脚本需要管理员权限。
4. **⚠️ 必须保存密码**：当你点击“确定”保存任务时，Windows 会弹出窗口要求你输入该用户的密码。**必须输入正确密码**，Windows 才能在后台缓存该用户的网络访问凭据（Logon Rights）。如果不输入密码，后台任务将无法访问任何网络资源。

### 🔎 如何进一步排查和调试？

如果按照上述方法修改后仍然失败，可以通过以下方式捕获错误信息：

1. **重定向脚本输出到日志文件**：
  在任务计划程序的“操作”选项卡中，修改你的程序/脚本调用方式，将标准输出和错误输出重定向到一个本地日志文件中，方便查看具体报错。
  - **程序或脚本**: `cmd.exe`
  - **添加参数**: `/c "C:\path\to\your_script.bat > C:\temp\task_log.txt 2>&1"`
   *(如果是 PowerShell，参数为: `-ExecutionPolicy Bypass -File "C:\path\to\script.ps1" > "C:\temp\task_log.txt" 2>&1`)*
2. **查看任务计划程序的历史记录**：
  在任务计划程序右侧的操作面板中，点击“启用所有任务历史记录”（如果未启用）。运行失败后，查看该任务的“历史记录”选项卡，通常会显示类似 `0x1` (常规错误) 或 `0x4` (内存不足/权限问题) 的错误代码。
3. **测试 UNC 路径访问**：
  以你配置在任务计划程序中的那个用户身份，打开一个新的 CMD 窗口（可以使用 `runas /user:YourUsername cmd.exe`），然后手动执行 `dir \\Server\Share`，确保该账户在不依赖映射驱动器的情况下，确实能直接访问该网络路径。

**总结**：放弃在后台任务中依赖手动创建的映射驱动器字母，**改用 UNC 路径 + 正确的用户账户及密码缓存**，是解决此问题最彻底的方法。
