# windows中有没有类似linux中的文件/目录软链接的功能

是的，Windows 中有与 Linux 非常类似的文件/目录软链接和硬链接功能。Windows 主要通过内置的命令行工具 `mklink` 来实现，它在 NTFS 文件系统级别创建链接，对绝大多数应用程序是完全透明的（这点与普通的 `.lnk` 快捷方式有本质区别）。

以下是详细的功能说明和使用指南：

### 1. `mklink` 命令的四种模式
`mklink` 的基本语法为：`mklink [[/D] | [/H] | [/J]] <链接名称> <目标路径>` [[19]]。

| 参数 | 链接类型 | 对应 Linux 概念 | 适用对象 | 是否可跨磁盘分区 | 目标被删除后的表现 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **无参数** | **符号链接** (Symbolic Link) | 软链接 (`ln -s`) | **文件** | 是 | 链接失效（悬空） |
| **`/D`** | **目录符号链接** | 目录软链接 | **目录** | 是 | 链接失效（悬空） |
| **`/H`** | **硬链接** (Hard Link) | 硬链接 (`ln`) | **文件** | **否** (必须同卷) | 无影响（数据仍存在） |
| **`/J`** | **目录联接** (Junction) | 类似目录硬链接 | **目录** | **否** (仅限本地卷) | 通常无影响 |

### 2. 核心特性对比
- **符号链接（软链接）**：与 Linux 的软链接完全一致。它存储的是目标的路径字符串，可以指向文件或目录，支持跨磁盘分区甚至网络路径。如果原始文件被删除，链接将失效 [[12]]。
- **硬链接**：与 Linux 的硬链接一致。它直接指向文件的物理数据（NTFS 中的 MFT 记录）。多个硬链接共享同一份数据，删除其中一个不会影响其他链接，只有当所有链接都被删除时，数据才会被真正释放 [[12]]。**注意：Windows 不支持目录的硬链接。**
- **目录联接 (Junction)**：这是 Windows 较早引入的特性，专门用于目录。它类似于目录软链接，但只能用于本地磁盘分区，不支持网络路径 [[13]]。它的系统级兼容性极好，某些旧版 Windows 程序对它的支持比符号链接更好。

### 3. 权限要求与“免管理员”技巧
- **默认情况**：创建符号链接通常需要**管理员权限**（需以管理员身份运行 CMD 或 PowerShell） [[2]]。
- **免管理员技巧（强烈推荐）**：在 Windows 10 (版本 1703 及以上) 和 Windows 11 中，如果系统开启了 **“开发者模式”**，普通用户无需管理员权限即可直接使用 `mklink` 创建符号链接 [[3]]。
  - *开启方法*：打开 `设置` -> `系统` (或 `系统和更新`) -> `开发者选项` -> 打开 `开发人员模式` 开关。

### 4. 实际操作示例
假设你有一个原始文件 `C:\Original\file.txt` 和原始目录 `C:\Original\Folder`，你想在 `D:\Link\` 下创建链接：

```cmd
:: 1. 创建【文件】软链接
mklink "D:\Link\file.txt" "C:\Original\file.txt"

:: 2. 创建【目录】软链接 (支持跨盘符)
mklink /D "D:\Link\Folder" "C:\Original\Folder"

:: 3. 创建【文件】硬链接 (注意：必须在同一个 C 盘内)
mklink /H "C:\Link\file_hard.txt" "C:\Original\file.txt"

:: 4. 创建【目录】联接 (Junction，仅限本地磁盘)
mklink /J "D:\Link\Folder_Junction" "C:\Original\Folder"
```
*(注意：命令中的路径如果包含空格，必须用双引号括起来)*

### 5. 与 Windows 普通“快捷方式 (.lnk)”的区别
- **快捷方式**：本质上只是一个普通的 `.lnk` 文件，需要 Windows 资源管理器或特定程序“解析”后才能跳转。很多命令行工具、编译器或专业软件无法识别它。
- **mklink 链接**：在 NTFS 文件系统底层实现。对操作系统和几乎所有应用程序来说，它**就是**真实的文件或目录，程序读写链接时会自动透明地重定向到目标路径，不会报错。

### 6. 图形化替代方案
如果你不习惯使用命令行，可以安装免费的第三方 Shell 扩展工具，例如 **Link Shell Extension**。安装后，只需：
1. 右键点击目标文件/文件夹，选择 `Pick Link Source`（选取链接源）。
2. 导航到目标位置，右键空白处选择 `Drop As` -> `Symbolic Link`（符号链接）或 `Hardlink`（硬链接）即可可视化完成操作。
