# 编写一个windows的批处理文件(bat文件），它根据当前时间的小时执行不同的任务。这个bat文件被任务管理器调度，每小时执行一次

这是一个专为 Windows 任务计划程序设计的批处理脚本。它已针对时区格式差异、任务计划程序的工作目录问题以及日志记录做了优化。

### 📜 批处理脚本 (`hourly_tasks.bat`)

```bat
@echo off
:: 切换到脚本所在目录，防止任务计划程序使用默认工作目录导致相对路径失效
cd /d "%~dp0"
setlocal

:: ================= 1. 获取当前小时 =================
:: 截取时间字符串的前两位（例如 " 9" 或 "14"）
set "raw_hour=%TIME:~0,2%"
:: 处理单数小时前可能存在的空格（Windows 单数小时默认带空格，如 9:00 显示为 " 9"）
if "%raw_hour:~0,1%"==" " set "raw_hour=0%raw_hour:~1%"
:: 转换为纯数字（去除前导0，例如 "08" -> 8，方便后续数值比较）
set /a hour=1%raw_hour% - 100

:: 日志文件路径（保存在脚本同级目录）
set "log_file=%~dp0hourly_task.log"

:: ================= 2. 记录执行日志 =================
echo [%date% %time%] 触发执行 | 当前小时: %hour% >> "%log_file%"

:: ================= 3. 任务分支逻辑 =================
:: 注意：bat 的 if 支持 geq(>=), leq(<=), gtr(>), lss(<), equ(==) 等比较符

if %hour% geq 6 if %hour% leq 8 (
    echo [06-08点] 执行早晨初始化任务...
    rem ★★★ 在此替换你的实际命令 ★★★
    rem 示例: call morning_job.bat
    rem 示例: python generate_report.py

) else if %hour% geq 9 if %hour% leq 17 (
    echo [09-17点] 执行日间同步任务...
    rem ★★★ 在此替换你的实际命令 ★★★
    rem 示例: robocopy "D:\Data" "E:\Backup" /MIR /LOG:sync.log

) else if %hour% geq 18 if %hour% leq 22 (
    echo [18-22点] 执行晚间备份任务...
    rem ★★★ 在此替换你的实际命令 ★★★
    rem 示例: 7z a daily_backup_%date:~0,4%%date:~5,2%%date:~8,2%.zip "D:\Work"

) else (
    echo [23-05点] 执行深夜维护/休眠任务...
    rem ★★★ 在此替换你的实际命令 ★★★
    rem 示例: taskkill /IM heavy_app.exe /F
    rem 示例: shutdown /s /t 300
)

:: ================= 4. 记录结束日志 =================
echo [%date% %time%] 执行完成 >> "%log_file%"
echo. >> "%log_file%"

endlocal
exit /b
```

### ⚙️ 任务计划程序配置步骤

1. **保存脚本**：将上述代码保存为 `C:\Scripts\hourly_tasks.bat`（路径可自定义）。
2. **打开任务计划程序**：`Win + R` 输入 `taskschd.msc` 回车。
3. **创建任务**：
   - **常规**：命名任务，勾选 `使用最高权限运行`（如果任务需要管理员权限）。
   - **触发器** → `新建`：
     - 开始任务：`按计划`
     - 设置：`每天`，具体时间设为 `00:00`（或你希望的首次运行时间）
     - ✅ 勾选 `重复任务间隔`：`1 小时`
     - ✅ 勾选 `持续时间`：`无限期`
   - **操作** → `新建`：
     - 操作：`启动程序`
     - 程序或脚本：`C:\Scripts\hourly_tasks.bat`（或直接填脚本完整路径）
     - 起始于（可选）：`C:\Scripts`
   - **条件**：建议取消勾选 `只有在计算机使用交流电源时才启动此任务`（防止笔记本断电不执行）。
   - **设置**：勾选 `如果任务失败，按以下频率重启`，并设置重试间隔。

### 💡 关键注意事项

| 问题 | 解决方案 |
|------|----------|
| **时间格式差异** | 脚本已处理 `TIME` 环境变量中单数小时的空格问题，并转换为纯数字，兼容所有 Windows 语言/区域设置。 |
| **路径失效** | `cd /d "%~dp0"` 确保无论任务计划程序以什么身份运行，所有相对路径都基于脚本所在目录。 |
| **日志查看** | 每次执行后会在同级目录生成/追加 `hourly_task.log`，方便排查任务是否按时触发及执行了哪个分支。 |
| **精确整点执行** | 若希望只在特定整点执行（如仅 9 点、14 点、20 点），可将条件改为：`if %hour% equ 9 ( ... ) else if %hour% equ 14 ( ... )` |
| **隐藏黑窗口** | 在任务计划的“操作”中，将程序改为 `cmd.exe`，参数填 `/c "C:\Scripts\hourly_tasks.bat"`，并在“条件”中勾选“隐藏”，可实现静默后台执行。 |

如有需要调整小时区间、添加错误重试逻辑或改为调用外部脚本，可直接修改 `rem ★★★` 下方的命令部分。
